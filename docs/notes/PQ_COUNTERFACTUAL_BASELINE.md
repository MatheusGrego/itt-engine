# P_eq as Geometric Counterfactual Baseline

> **Status:** Visually validated, implementation-pending.
> **Relates to:** Open Question H1 (roadmap).
> **Date:** 2026-04-25.

## TL;DR

Replace `P_eq = uniform maximum entropy` with `P_eq = the same system
without the entity under analysis`. Validated visually on a 2D black hole
ray-tracer; the τ peak coincides with the photon sphere only under this
formulation. Generalizes to networks via local counterfactual rewiring.

---

## Context

Insight emerged during informal visual exploration — a 2D
Schwarzschild-like ray-tracer was built as a "play" widget to see ITT
applied to a continuous physical system rather than a discrete network.

The initial implementation used a directional-disagreement proxy for τ
(JSD over local ray direction distributions). It produced a clear but
**asymmetric** τ field, biased downstream of the black hole rather than
peaking at the geometrically meaningful boundary (photon sphere).

This is structurally the same problem already articulated in the
roadmap: when P_exp depends on observer choice or implicit assumption,
τ inherits that bias. The visual format made the failure mode obvious
in seconds — something the formalism took months to fully expose.

## The Reformulation

Rather than `P_eq = uniform` (max entropy with no constraints), or
`P_exp = some modeled prior`, define:

> **P_eq = the counterfactual distribution generated by the same system
> with the entity-of-interest (or the perturbation source) removed.**

In the black hole case this is operational and clean:

- **P_obs:** density field of light rays under gravity
- **P_eq:** density field of "ghost rays" with identical initial
  conditions but **no gravity** (straight lines)
- **τ(cell) = JSD(P_obs(cell) || P_eq(cell))**

Properties of this P_eq:

1. **Observer-independent** — defined by the system's own counterfactual,
   not by external choice of distribution
2. **Locally meaningful** — computed pointwise, not globally assumed
3. **Empirically grounded** — no model assumptions beyond "remove the
   perturbation source"

## Why this resolves Open Question H1

H1 (paraphrased): *Is τ ontologically meaningful, or just an
epistemological tool dependent on observer choice of P_exp?*

Counterfactual P_eq makes τ ontological in a stronger sense than
uniform max-ent did:

- Uniform max-ent answers: *"different from a totally featureless system"*
- Counterfactual answers: *"different from this system without this entity"*

The second is what *"tension caused by X"* actually means. The first
was a proxy that worked when local structure was approximately uniform,
but degraded wherever structure was non-trivial — which is exactly the
interesting cases.

## Generalization to Networks

For graph G and node v under analysis:

```
P_obs(v) = observed distribution of [property] on N(v) in G
P_eq(v)  = observed distribution of [property] on N(v) in G',
           where G' = G with v's edges rewired under a configuration
                model preserving local degree distribution
τ(v)     = JSD(P_obs(v) || P_eq(v))
```

Two variants worth comparing:

- **Strict counterfactual:** G' = G with v fully removed
- **Soft counterfactual:** G' = G with v's edges rewired (configuration
  model) — preserves degree, removes structural signature

Computationally heavier than uniform max-ent, but still O(|N(v)|) per
node since rewiring is local.

## Visual Validation

`pq_baseline_demo.html` (this commit) implements the reformulation on
the 2D toy black hole. Three viewable layers:

- **P_obs** — actual light rays bent by gravity
- **P_eq** — ghost rays without gravity, same initial conditions
- **τ** — pointwise divergence between the two density fields

Result: with counterfactual P_eq the τ peak coincides with the photon
sphere (r ≈ 1.5 × r_horizon in this toy metric). With the previous
direction-based proxy, τ peaked downstream and asymmetrically.

This is **visual** validation, not statistical. It confirms the
reformulation produces geometrically meaningful results in a system
with known ground truth, but is not a replacement for empirical testing
on real networks.

## What This Does and Does Not Validate

**Validates:**
- Reformulation produces geometrically meaningful τ fields
- Visual intuition matches the formal claim
- The counterfactual definition is implementable

**Does not validate:**
- Empirical performance on existing validation cases (NPM, SPARC,
  Marak) — needs porting and re-running
- Computational tractability at the engine's current throughput
  (~7.6M events/s) — counterfactual graphs are heavier
- That counterfactual is the *right* P_eq vs alternatives (ER baseline,
  pure configuration model, edge-swapped null) — comparative testing
  needed

## Implementation Notes

The engine currently uses uniform max-ent baseline (per `Builder` default).
Adding counterfactual P_eq is non-trivial because:

1. Computing P_eq for node v requires local graph manipulation, not
   just a distribution lookup
2. Caching strategies need updating — P_eq is now node-specific
3. MAD calibrator was tuned against uniform-baseline τ distributions;
   likely needs re-calibration for counterfactual-baseline distributions
4. Throughput impact unknown — needs benchmark before committing

**Suggested architectural approach:**

- Introduce a `BaselineStrategy` interface for P_eq computation
- Keep `UniformMaxEntropyBaseline` as default (backwards compatibility,
  preserves existing benchmarks)
- Add `CounterfactualBaseline` as opt-in via Builder
- Run regression suite on existing validation cases (Marak, SPARC, NPM)
  comparing both baselines side-by-side
- Decide based on (false-positive rate × computational cost) tradeoff
